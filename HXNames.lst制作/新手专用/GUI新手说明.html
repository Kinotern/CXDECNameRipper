HXv4 HxNames GUI 新手说明

这个 GUI 用来生成 HxNames.lst，并把 HXv4 hash 文件名和目录名尽量还原成可读名称。

第一步 解包游戏 xp3。
第二步 双击 启动GUI.bat。
第三步 选择目录、勾选来源、开始生成。

推荐流程

  1. 先用 KrkrExtractForCxdecV2 或 GARBro2 把游戏的 xp3 解包到一个目录。
  2. 双击项目目录里的 启动GUI.bat。
  3. 在“待处理/重命名目录”选择刚才解包出来的总目录。
  4. HxNames.lst 默认会生成在项目目录，也可以自己选择保存位置。
  5. 在“明文字典来源”里按你已有的文件勾选来源。
  6. 保持“生成后自动按 HxNames 重命名目录和文件”勾选。
  7. 点击“生成 HxNames 并重命名”，然后看底部运行日志。

来源怎么选

  • 有 scn 剧本目录：优先勾选“扫描 scn/PSB 剧本目录”。
  • 有同游戏体验版、旧版、未加密版：勾选“从明文目录收集文件名”。
  • 有 base.stage、cglist.csv、soundlist.csv、charvoice.csv 等文件：分别勾选对应项。
  • 不确定时，先只选“扫描 scn/PSB 剧本目录”和“从明文目录收集文件名”。
勾选了某个来源后，一定要在右边选择对应文件或目录。否则 GUI 会在日志里提示跳过。

其他按钮

  • 恢复目录结构：GARBro 把目录 hash 和文件 hash 拼在一个文件名里时使用，例如 目录hash_文件hash.ext。
  • 生成干净 HxNames：发布给别人用之前，从已经反混淆过的目录里筛出实际命中的条目，减少冗余。

常见问题

  • 第一次运行会安装 json5，需要能访问 pip。
  • PsbDecompile.exe、pbd2json.exe、KrkrHxv4Hash.dll 必须保留在项目自带的 binaries 目录里。
  • 如果日志里提示路径不存在，检查是否勾选了来源但没有选择文件/目录。
  • 如果还原率低，通常需要更多明文来源，比如旧版/体验版解包目录、CSV 列表、KrkrDump 日志。
建议先备份解包目录再执行自动重命名。GUI 会尽量避免覆盖同名文件，但备份能避免误选目录带来的麻烦。
